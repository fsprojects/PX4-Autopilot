## Purpose

`math::constrain(val, min_val, max_val)` clamps a value to a closed interval
`[min_val, max_val]`. It is a ubiquitous utility used throughout PX4 flight control
to keep values within safe operating bounds (e.g., throttle in [0, 1], angular rate
limits, battery voltage bounds).

## C++ Implementation

```cpp
template<typename _Tp>
constexpr _Tp constrain(_Tp val, _Tp min_val, _Tp max_val)
{
    return (val < min_val) ? min_val : ((val > max_val) ? max_val : val);
}
```

## Preconditions

- `min_val ≤ max_val` — the interval must be non-empty (otherwise the result is
  implementation-defined/meaningless)
- For numeric types: no NaN inputs (float version); integer types have no preconditions
  beyond the above

## Postconditions

1. **In-range**: `min_val ≤ result ≤ max_val` (given `min_val ≤ max_val`)
2. **Fixed point for in-range values**: if `min_val ≤ val ≤ max_val`, then
   `result = val`
3. **Lower clamp**: if `val < min_val`, then `result = min_val`
4. **Upper clamp**: if `val > max_val`, then `result = max_val`

## Derived Properties

- **Idempotence**: `constrain(constrain(v, lo, hi), lo, hi) = constrain(v, lo, hi)`
  (applying constrain twice gives the same result as once)
- **Monotonicity**: if `v1 ≤ v2`, then `constrain(v1, lo, hi) ≤ constrain(v2, lo, hi)`
- **Non-shrinking lower bound**: `constrain(v, lo, hi) ≥ lo` always
- **Non-exceeding upper bound**: `constrain(v, lo, hi) ≤ hi` always
- **Composition**: `constrain(constrain(v, lo1, hi1), lo2, hi2)` equals
  `constrain(v, max(lo1, lo2), min(hi1, hi2))` when bounds are compatible

## Edge Cases

- `min_val = max_val`: returns `min_val` for any input
- `val = min_val`: returns `min_val`
- `val = max_val`: returns `max_val`
- Floating point: if `val` is NaN, the behaviour is implementation-defined
  (both comparisons `val < min` and `val > max` are false for NaN, so NaN would
  be returned — this is a known PX4 edge case documented in `isFinite` helpers)
- Very large values: no overflow since we return one of the three inputs unchanged
