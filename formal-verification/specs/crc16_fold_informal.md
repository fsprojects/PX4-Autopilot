## Purpose

Computes a CRC-16 checksum over a byte buffer. The checksum is used to verify the
integrity of SBF (Septentrio Binary Format) GNSS receiver packets. This specific
algorithm implements the CCITT variant using the generator polynomial 0x1021,
matching the checksum specified by the Septentrio SBF protocol.

The function processes each byte iteratively, updating a 16-bit running checksum
state using bitwise XOR and shift operations.

## Preconditions

1. `data_p` is a valid pointer to a contiguous array of at least `length` bytes.
   (In the Lean model, this is abstracted as a `List UInt8`.)
2. `length ≥ 0` (vacuously satisfied since `length` is `uint32_t`).

No preconditions on the content of the data buffer.

## Postconditions

1. **Returns a `uint16_t`**: the result is in the range `[0, 65535]`.
2. **Empty buffer**: `buffer_crc16([], 0) = 0`.
   (The initial CRC state is 0, and no bytes are processed.)
3. **Deterministic**: the same byte sequence always produces the same CRC.
4. **Fold/split (incremental computation)**:
   For any two buffers `a` and `b`:
   ```
   buffer_crc16(a ++ b) = crc16_continue(buffer_crc16(a), b)
   ```
   where `crc16_continue(initial_crc, buffer)` is the same algorithm with a
   non-zero starting CRC instead of 0. This is the key algebraic property of all
   CRC algorithms: the computation is a left fold over the input bytes.
5. **Single byte**: `buffer_crc16([x]) = crc16_step(0, x)` for any byte `x`,
   where `crc16_step` is the per-byte update function.

## Invariants

- The function is a **homomorphism from byte lists to CRC values** under the
  `crc16_continue` composition:
  `crc16_continue(crc16(a), b) = crc16(a ++ b)`
- The computation is equivalent to `List.foldl crc16_step 0 bytes`.

## Algorithm Details

The per-byte update function (extracted from the C++ loop body):

```cpp
// Given current crc and new byte b:
uint8_t  x   = (crc >> 8) ^ b;
x           ^= (x >> 4);
uint16_t new_crc = (crc << 8) ^ (x << 12) ^ (x << 5) ^ x;
```

This corresponds to the CRC-16/CCITT-0 (also called CRC-16/KERMIT or CRC-16/CCITT
FALSE) algorithm with polynomial 0x1021, processed MSB-first.

## Examples

Let `crc16_step(crc, b)` denote the per-byte update. Then:
- `crc16([]) = 0`
- `crc16([b]) = crc16_step(0, b)`
- `crc16([b1, b2, b3]) = crc16_step(crc16_step(crc16_step(0, b1), b2), b3)`
- `crc16(a ++ b) = crc16_continue(crc16(a), b)`

## Inferred Intent

The function is used by the Septentrio GNSS driver to validate incoming SBF packet
checksums. The incremental/fold property is essential for validating packets that
arrive in chunks: the driver could (in principle) update the running CRC as each
byte arrives rather than waiting for the complete packet.

The most safety-critical property: **if the CRC matches, data corruption is
detected with high probability** (false acceptance probability ≈ 1/65536 for random
errors, lower for burst errors with the CCITT polynomial).
