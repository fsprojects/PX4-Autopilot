# Informal Specification: `math::interpolateNXY`

🔬 *Generated by Lean Squad automated formal verification.*

## Source

`src/lib/mathlib/math/Functions.hpp`, template function `interpolateNXY<T, N>`.

```cpp
template<typename T, size_t N>
const T interpolateNXY(const T &value, const T(&x)[N], const T(&y)[N])
{
    size_t index = 0;
    while ((value > x[index + 1]) && (index < (N - 2))) {
        index++;
    }
    return interpolate(value, x[index], x[index + 1], y[index], y[index + 1]);
}
```

## Purpose

A **piecewise-linear interpolation** over an arbitrary sorted N-point table.
Given:
- A query value `v`,
- Sorted breakpoint array `x[0] < x[1] < ... < x[N-1]`,
- Corresponding output array `y[0], y[1], ..., y[N-1]`,

The function finds the segment `[x[i], x[i+1]]` containing `v` and returns the
linear interpolation of the corresponding outputs `y[i]` and `y[i+1]`.
Outside the range `[x[0], x[N-1]]`, the result is clamped to the boundary segment.

## Usage in PX4

Used for sensor calibration curves, RC stick mappings, and actuator output scaling
where a piecewise-linear response is configured by N control points (e.g.,
`MC_AIRSPEED_SCALE` lookup tables, motor efficiency curves).

## Preconditions

1. `N ≥ 2` (at least one segment).
2. Breakpoints strictly increasing: `x[0] < x[1] < ... < x[N-1]`.
3. `value` is any finite real number (NaN/infinity are not modelled).

## Postconditions

1. **Segment selection**: the returned index `i` satisfies:
   - `i = 0` if `value ≤ x[1]`,
   - `i = k` if `x[k] < value ≤ x[k+1]` for `1 ≤ k ≤ N-3`,
   - `i = N-2` if `value > x[N-2]`.

2. **Low clamp**: if `value ≤ x[0]`, the output equals `y[0]`
   (the inner `interpolate` call clamps at the low end).

3. **High clamp**: if `value > x[N-1]`, the output equals `y[N-1]`
   (the inner `interpolate` call clamps at the high end).

4. **Continuity at breakpoints**: at `value = x[i]` for any interior breakpoint `i`,
   the result equals `y[i]` (the segments join continuously).

5. **Range containment**: if the y-values are monotone non-decreasing
   (`y[0] ≤ y[1] ≤ ... ≤ y[N-1]`), the output lies in `[y[0], y[N-1]]`.

6. **Endpoint correctness**:
   - `interpolateNXY(x[0], x, y) = y[0]`
   - `interpolateNXY(x[N-1], x, y) = y[N-1]`

## Invariants

- Output is always in the range `[min(y), max(y)]` when y-values are monotone.
- The piecewise-linear function is continuous on `(-∞, +∞)`.

## Edge Cases

- `N = 2`: degenerates to a single `interpolate` call between `x[0]` and `x[1]`.
- `value = x[i]` exactly (breakpoint): both adjacent segments give `y[i]`
  (well-defined due to the strict `>` condition in the while loop).
- Non-monotone y-arrays: the function still runs but may not be monotone or
  have a bounded range.

## Examples (3-point, N=3)

```
x = [0, 1, 2], y = [0, 5, 10]
interpolateNXY(0)    = 0      (at first breakpoint)
interpolateNXY(0.5)  = 2.5    (first segment, midpoint)
interpolateNXY(1.0)  = 5.0    (at breakpoint x[1], second segment gives 5)
interpolateNXY(1.5)  = 7.5    (second segment, midpoint)
interpolateNXY(2.0)  = 10.0   (at last breakpoint)
interpolateNXY(-1.0) = 0      (clamped low)
interpolateNXY(3.0)  = 10.0   (clamped high)
```

## Inferred Intent

The `>` condition (not `≥`) in the while loop means the left breakpoint `x[i]` is
**included** in segment `i`, and the right breakpoint `x[i+1]` is included in
segment `i+1`. This ensures well-defined behaviour at interior breakpoints:
`value = x[i]` uses segment `i-1` (not segment `i`), but since the segments join
continuously at `y[i]`, the output is the same regardless.

## Open Questions

1. **Overflow**: `(T)(index + 1)` could overflow for very large N. In practice, N is
   small (≤ 10) and T is float, so this is not a concern.
2. **Equal x-values**: the precondition requires strictly increasing x-values, but the
   C++ code does not enforce this. If `x[i] = x[i+1]`, `interpolate` will divide by
   zero. This should be documented as a precondition violation.
3. **Non-finite values**: NaN propagation through `interpolate` is not specified.
