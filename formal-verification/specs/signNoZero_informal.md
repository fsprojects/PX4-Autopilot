## Purpose

`math::signNoZero(val)` returns the sign of a value as `+1` or `-1`, treating
zero as positive (non-negative). Unlike the standard mathematical sign function
(which returns 0 for zero input), this variant guarantees the result is always ±1.
This is used in contexts where the result will be used as a multiplier and zero
would be incorrect (e.g., control direction, normalisation).

## C++ Implementation

```cpp
template<typename T>
int signNoZero(T val)
{
    return (T(0) <= val) - (val < T(0));
}
```

The arithmetic trick: `(0 <= val)` is 1 if non-negative, 0 otherwise.
`(val < 0)` is 1 if negative, 0 otherwise. Their difference is:
- `1 - 0 = 1` if `val >= 0`
- `0 - 1 = -1` if `val < 0`

## Preconditions

- None — defined for all values of type `T`
- For integer types: all inputs including `INT_MIN` are valid

## Postconditions

1. **Binary output**: result ∈ {-1, 1} always
2. **Correct for non-negative**: if `0 ≤ val`, then `result = 1`
3. **Correct for negative**: if `val < 0`, then `result = -1`
4. **Never zero**: result ≠ 0

## Derived Properties

- `signNoZero(0) = 1`
- `signNoZero(-1) = -1`
- `|signNoZero(v)| = 1` always
- `signNoZero(v) * signNoZero(-v) = -1` for all `v ≠ 0`
- `signNoZero(v) * signNoZero(v) = 1` always (idempotence of multiplication)

## Edge Cases

- `val = 0`: returns `1` (zero treated as positive)
- `val = INT_MIN`: returns `-1` (large negative)
- `val = INT_MAX`: returns `1`
