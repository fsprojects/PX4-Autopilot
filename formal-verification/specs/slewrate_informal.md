## Purpose

`SlewRate::update(new_value, deltatime)` is a rate limiter: it moves the current
tracked value toward `new_value` but limits the rate of change per unit time. If the
desired change exceeds `slew_rate * deltatime`, the output is clamped to the maximum
allowed change. Used throughout PX4 to protect actuators from abrupt commands and to
smooth setpoints.

## C++ Implementation

```cpp
Type update(const Type new_value, const float deltatime)
{
    const Type dvalue_desired = new_value - _value;
    const Type dvalue_max = _slew_rate * deltatime;
    const Type dvalue = math::constrain(dvalue_desired, -dvalue_max, dvalue_max);
    _value += dvalue;
    return _value;
}
```

## Preconditions

- `deltatime > 0` — time step must be positive
- `slew_rate ≥ 0` — rate limit must be non-negative
- The state `_value` is initialised (via constructor or `setForcedValue`)

## Postconditions

1. **Rate bound**: `|result - prev_value| ≤ slew_rate * deltatime`
2. **Exact tracking when reachable**: if `|new_value - prev_value| ≤ slew_rate * deltatime`,
   then `result = new_value`
3. **Direction**: if `new_value > prev_value`, then `result ≥ prev_value` (non-decreasing
   when target is above)
4. **Direction**: if `new_value < prev_value`, then `result ≤ prev_value` (non-increasing
   when target is below)
5. **State update**: the internal `_value` is updated to `result` after each call

## Derived Properties

- `update(prev_value, dt) = prev_value` (no change when already at target)
- The sequence of outputs converges monotonically to `new_value` if it is held constant

## Edge Cases

- `deltatime = 0`: `dvalue_max = 0`, so `result = _value` (no change)
- `slew_rate = 0`: output never changes, always returns current state
- Very large `new_value`: the output increases by `slew_rate * deltatime` per step

## Examples

State `_value = 0`, `slew_rate = 1.0`:
| `new_value` | `deltatime` | `result` |
|-------------|-------------|----------|
| 0.5         | 1.0         | 0.5      |
| 2.0         | 1.0         | 1.0      |
| -0.5        | 1.0         | -0.5     |
| -2.0        | 1.0         | -1.0     |

## Relationship to `constrain`

`SlewRate::update` directly calls `math::constrain` to clamp `dvalue`. Proving
the rate bound postcondition reduces to proving `constrain`'s in-range property.
This is why `constrain` was targeted first.
