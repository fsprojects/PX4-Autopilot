# Informal Specification: `math::superexpo`

🔬 *Generated by Lean Squad automated formal verification.*

**Source**: `src/lib/mathlib/math/Functions.hpp`

---

## Purpose

`superexpo(value, e, g)` is an enhanced RC stick curve function used in PX4's flight
controller. It augments the basic `expo` curve (controlled by parameter `e`) with a
"superrate" term (controlled by parameter `g`) that dramatically increases sensitivity
near full deflection. This allows pilots to have both a soft centre feel and very high
maximum rates at full stick throw.

The C++ template implementation:

```cpp
template<typename T>
const T superexpo(const T &value, const T &e, const T &g)
{
    T x = math::constrain(value, (T) - 1, (T)1);
    T gc = math::constrain(g, (T)0, (T)0.99);
    return expo(x, e) * (1 - gc) / (1 - fabsf(x) * gc);
}
```

---

## Preconditions

- `value`: RC stick input in `[-1, 1]` (clamped internally if out of range)
- `e`: expo factor; meaningful range is `[0, 1]`; values outside are accepted
- `g`: superrate factor; meaningful range is `[0, 0.99]`; values outside are clamped to `[0, 0.99]`

---

## Postconditions

1. **Range containment**: output ∈ `[-1, 1]`
2. **Zero fixed point**: `superexpo(0, e, g) = 0`
3. **Unit fixed points**: `superexpo(1, e, g) = 1` and `superexpo(-1, e, g) = -1`
4. **Odd symmetry**: `superexpo(-v, e, g) = -superexpo(v, e, g)`
5. **Reduction to expo**: when `g = 0`, `superexpo(v, e, 0) = expo(v, e)`
6. **Denominator safety**: `1 - |x| * gc > 0` for all clamped `x` and `gc`, so the
   division is never by zero

---

## Invariants

- The denominator `(1 - |x| * gc)` is always strictly positive because:
  - `|x| ≤ 1` (after clamping)
  - `gc ≤ 0.99` (after clamping)
  - Therefore `|x| * gc ≤ 0.99 < 1`
- The superrate factor `g` is clamped to `[0, 0.99)` to prevent division by zero at
  full stick deflection (`|x| = 1`)

---

## Edge Cases

| Input | Expected output |
|-------|----------------|
| `value = 0` | `0` (regardless of `e` and `g`) |
| `value = 1` | `1` (regardless of `e` and `g`) |
| `value = -1` | `-1` (regardless of `e` and `g`) |
| `g = 0` | `expo(value, e)` — superrate disabled |
| `g = 0.99` | Maximum superrate; output ≈ `expo(value, e) / (1 - |value| * 0.99)` |
| `value > 1` | Clamped to `1` before processing |
| `value < -1` | Clamped to `-1` before processing |
| `g > 0.99` | Clamped to `0.99` |
| `g < 0` | Clamped to `0` (i.e., pure expo) |

## Examples

```
superexpo(0,    0.5, 0.5) = 0
superexpo(1,    0.5, 0.5) = 1
superexpo(-1,   0.5, 0.5) = -1
superexpo(0.5,  0,   0  ) = 0.5   (linear, no expo, no superrate)
superexpo(0.5,  0,   0.5) = 0.5 * 1 / (1 - 0.25) = 0.5/0.75 ≈ 0.667
```

## Inferred Intent

The function is designed to give a **non-linear** mapping from stick position to
output that:
- Is **soft in the centre** (low sensitivity near zero): controlled by `expo`
- Is **aggressive at the edges** (high sensitivity near ±1): amplified by `g`
- Always **maps endpoints to endpoints** (1 → 1, -1 → -1, 0 → 0) for predictable feel
- Is **odd** (symmetric around zero) so the aircraft responds identically in both
  directions

The `gc` clamping to `[0, 0.99]` is a safety guard: if `g` were exactly 1 and `|x|`
were exactly 1, the denominator would be zero. The 0.99 cap prevents this.
